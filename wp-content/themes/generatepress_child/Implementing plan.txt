│ Episode Interactions System — Implementation Plan                            │
│                                                                              │
│ Context                                                                      │
│                                                                              │
│ Episode pages currently have no user interaction features. Users want to     │
│ rate, like/dislike, comment on individual episodes, and track which episodes │
│  they've watched. The fanfiction plugin (on a separate site) already         │
│ implements ratings/likes/localStorage persistence patterns that we'll adapt  │
│ — not import — into the SMPT mu-plugin.                                      │
│                                                                              │
│ Episodes are NOT WP posts. They're rendered by gerar_episodios() in          │
│ tabelas_episodios.php as HTML cards with IDs like episodio_001. This means   │
│ we need custom DB tables for interactions and comments (can't use WP post    │
│ meta or native comments for per-episode data).                               │

│ 2. PHP Backend (episode-interactions.php)                                    │
│                                                                              │
│ Core Functions (adapted from fanfiction plugin patterns)                     │
│                                                                              │
│ - smpt_ep_hash_anon_uuid($uuid) — hash_hmac('sha256', $uuid,                 │
│ wp_salt('auth'), true)                                                       │
│ - smpt_ep_resolve_actor($user_id, $anon_uuid) — returns ['user_id' => int]   │
│ or ['anon_hash' => binary]                                                   │
│ - smpt_ep_set_reaction($ep_id, $user_id, $anon_uuid, $intent) — like/dislike │
│  state machine with advisory lock + transaction. Intents: like, remove_like, │
│  dislike, remove_dislike. Updates counters atomically.                       │
│ - smpt_ep_set_rating($ep_id, $user_id, $anon_uuid, $value) — upsert rating,  │
│ update counter sum/count                                                     │
│ - smpt_ep_remove_rating($ep_id, $user_id, $anon_uuid) — delete rating,       │
│ adjust counters                                                              │
│ - smpt_ep_set_watched($ep_id, $user_id) — logged-in only, upsert watched row │
│ - smpt_ep_remove_watched($ep_id, $user_id) — delete watched row              │
│ - smpt_ep_add_comment($ep_id, $user_id, $anon_uuid, $author_name, $text) —   │
│ insert + increment counter                                                   │
│ - smpt_ep_get_comments($ep_id, $page, $per_page) — paginated, oldest first   │
│ - smpt_ep_get_batch_counters($ep_ids) — single query for all episodes on     │
│ page                                                                         │
│ - smpt_ep_get_user_states($actor, $ep_ids) — batch load current user/anon    │
│ state                                                                        │
│ - smpt_ep_get_watched_episodes($user_id) — all watched episode_ids for       │
│ dashboard                                                                    │
│ - smpt_ep_reattribute_anon($user_id, $anon_hash) — on login, migrate anon    │
│ rows (delete dupes, update remainder)                                        │

│ 3. JavaScript (episode-interactions.js)                                      │
│                                                                              │
│ jQuery IIFE, loaded in footer, depends on jquery + smpt-analytics.           │
│                                                                              │
│ LocalStore (adapted from FanficLocalStore)                                   │
│                                                                              │
│ - Storage key: 'smpt_ep_interactions'                                        │
│ - Reuses window.smptVisitorHash from analytics.js as anonymous UUID (no      │
│ separate UUID needed)                                                        │
│ - Entry format: { 'ep_42': { like: true, rating: 4.5, watched: true,         │
│ timestamp: ms } }                                                            │
│ - Methods: get(epId), set(epId, entry), toggleLike(epId),                    │
│ toggleDislike(epId), setRating(epId, val), setWatched(epId, bool), getAll()  │
│                                                                              │
│ Page Init Flow                                                               │
│                                                                              │
│ 1. Collect episode IDs from .contentor_episodio elements on page             │
│ 2. GET /ep-stats?episodes=1,2,...46 → batch load counters + user states      │
│ 3. Merge server states with localStorage (server wins for logged-in,         │
│ localStorage wins for anon)                                                  │
│ 4. Render interaction bar into each .smpt-ep-interactions placeholder        │
│ 5. If config.needsSync → trigger login sync                                  │
│                                                                              │
│ UI Rendering (injected into .smpt-ep-interactions div)                       │
│                                                                              │
│ Each episode card gets an interaction bar containing:                        │
│ [★★★★☆ 4.2 (15)] [👍 12] [👎 2] [💬 3] [✓ Visto]                             │
│                                                                              │
│ - Stars: 5 SVG stars with half-star hit zones (same pattern as fanfiction).  │
│ Hover preview, click submits.                                                │
│ - Like/Dislike: Toggle buttons with counts. Mutually exclusive.              │
│ - Comments toggle: Shows count, expands comment section below when clicked.  │
│ - Watched: Checkmark badge (logged-in only).                                 │
│                                                                              │
│ Comments Section (lazy-loaded)                                               │
│                                                                              │
│ - Expand on click → GET /ep-comments/{id}?page=1                             │
│ - Shows author name + relative timestamp + text                              │
│ - Comment form: name field (pre-filled if logged in, editable if anon) +     │
│ textarea + submit button                                                     │
│ - Paginated with "load more" button                                          │
│                                                                              │
│ Optimistic UI Pattern                                                        │
│                                                                              │
│ 1. Update localStorage immediately                                           │
│ 2. Update DOM counts/states immediately                                      │
│ 3. POST to REST async                                                        │
│ 4. On response: correct counts from server stats object                      │
│ 5. On failure: revert localStorage + DOM                                     │
│                                                                              │
